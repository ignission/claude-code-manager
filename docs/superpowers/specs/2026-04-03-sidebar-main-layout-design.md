# サイドバー + メイン + Beacon 3カラムレイアウト設計

## 概要

現在の2列グリッドレイアウト（MultiPaneLayout）を廃止し、**サイドバー（セッション一覧） + メイン（ttydターミナル1ペイン） + Beacon（チャット）** の3カラムレイアウトに移行する。

## 背景・動機

- 4ペイン以上のグリッド表示では、ペイン間の狭い隙間でしかページスクロールできず操作性が悪い
- リポジトリ選択→worktree選択→セッション起動という階層的フローが煩雑
- PCでBeaconがダイアログ表示のため、ターミナル操作中にチャットを参照できない

## レイアウト構成

### PC表示

```
┌───────────┬──────────────────────┬──────────────┐
│ サイドバー  │  メイン (ttyd)        │  Beacon      │
│ (~250px)   │  (flex-1)            │  (~350px)    │
│            │                      │              │
│ [+ 新規]   │                      │  CRT風チャット│
│            │  選択中セッションの   │              │
│ 🟢 ark/    │  ターミナル           │  指示入力     │
│   feat-auth│  フルサイズ           │  クイック     │
│   Running..│                      │  コマンド     │
│            │                      │              │
│ 🔴 ark/    │                      │  履歴表示     │
│   fix-login│                      │              │
│   Ready    │                      │              │
│            │  [入力バー]           │  [入力欄]    │
└───────────┴──────────────────────┴──────────────┘
```

- **サイドバー** (~250px固定): 全セッションをフラット表示
- **メイン** (flex-1): 選択中セッションのttyd iframe + 入力バー
- **Beacon** (~350px固定): MobileChatViewを常設表示

### モバイル表示

既存のMobileLayoutをそのまま維持（タブ切り替え: セッション / Beacon）。

## サイドバー設計

### セッションカード

各セッションは以下の情報を表示する:

```
┌──────────────────────┐
│ 🟢 ark/feat-auth   ◀ │  ← 状態アイコン + repo短縮名/ブランチ + 選択中マーク
│   Running tests...    │  ← プレビュー（最新1行）
└──────────────────────┘
```

- **状態アイコン**: 🟢動作中 / 🔴停止（判断待ち） / 🟡idle
- **ラベル**: `{リポジトリ短縮名}/{ブランチ名}` 形式
- **選択マーク**: メインに表示中のセッションを`◀`等で明示
- **プレビューテキスト**: tmux capture-paneで取得した最新1行（ANSIエスケープ除去）

### 「+」ボタン（新規セッション作成）

サイドバー上部に配置。クリックでダイアログを開き:
1. リポジトリ選択（既存のRepoSelectDialog流用）
2. Worktree選択 or 新規作成（既存のCreateWorktreeDialog流用）
3. セッション起動

### セッション操作

- **クリック**: メインにそのセッションを表示
- **右クリック/長押し**: コンテキストメニュー（停止、削除等）

## プレビュー取得の仕組み

### サーバー側

`TmuxManager`に`capturePane`メソッドを追加:
- `tmux capture-pane -p -t <sessionName> -S -3` で最新3行を取得
- ANSIエスケープシーケンスを除去してプレーンテキストを返す

`SessionOrchestrator`にプレビューポーリングを追加:
- 3秒間隔で全アクティブセッションのプレビューを取得
- メインに表示中のセッションはスキップ（ttydで直接見えるため）

### Socket.IOイベント

新規イベント:
- `session:preview` (サーバー→クライアント): `{ sessionId: string, text: string, timestamp: number }`
- `session:previews` (サーバー→クライアント): `Array<{ sessionId, text, timestamp }>` （接続時の一括送信）

### パフォーマンス

- `tmux capture-pane`はtmuxのメモリ内バッファ読み取りで軽量
- セッション数×3秒間隔のポーリングは10セッション程度なら問題なし
- メイン表示中のセッションはスキップすることで不要な取得を回避

| コンポーネント | 責務 |
|---|---|
| `SidebarMainLayout.tsx` | 3カラムレイアウトのコンテナ。サイドバー+メイン+Beaconの配置 |
| `SessionSidebar.tsx` | サイドバー全体（セッション一覧 + 「+」ボタン） |
| `SessionCard.tsx` | 各セッションのカード（状態・ラベル・プレビュー） |

### 廃止

| コンポーネント | 理由 |
|---|---|
| `MultiPaneLayout.tsx` | グリッドレイアウト→3カラムに置き換え |
| `SessionDashboard.tsx` | リポジトリ/worktree選択画面→サイドバーの「+」ダイアログに移行 |

### 変更

| コンポーネント | 変更内容 |
|---|---|
| `Dashboard.tsx` | レイアウト切り替え。MultiPaneLayout→SidebarMainLayout。Beaconダイアログ削除（右カラムに常設）。状態管理の簡素化 |
| `TerminalPane.tsx` | 最大化ボタン削除（常に1ペインフル表示のため不要） |

## 状態管理

### Dashboard.tsx の状態変更

**追加:**
- `selectedSessionId: string | null` — メインに表示中のセッションID

**削除（不要に）:**
- `maximizedPane` — 常に1ペインのため
- `activePanes` / `activePanesPerRepo` — グリッド管理不要
- `closedPanes` — ペイン開閉の概念がなくなる
- `showBeaconDialog` — 常設表示のため

**維持:**
- `sessions: ManagedSession[]` — 全セッション
- `selectedRepo` — 「+」ダイアログでの新規作成時に使用
- beacon関連状態（`beaconMessages`, `beaconStreaming`等）

### 永続化

- `localStorage`に`selectedSessionId`を保存（リロード時に復元）
- `activePanesPerRepo`、`closedPanes`等のlocalStorageキーはクリーンアップ

## セッション選択ロジック

1. **サイドバーのカードクリック** → `selectedSessionId`更新 → メインのttyd iframeが切り替わる
2. **新規セッション作成** → 作成完了後、自動的にそのセッションを選択
3. **選択中セッション停止** → そのまま表示を維持（停止状態として表示）
4. **初回ロード・セッションなし** → メインエリアに空状態メッセージ（「+」ボタンで作成を促す）

## テスト方針

- サイドバーのセッション一覧表示・クリック切り替えの動作確認
- tmux capture-paneによるプレビュー取得の動作確認
- 3カラムレイアウトのレスポンシブ確認（ウィンドウ幅狭い場合）
- 既存機能の回帰テスト（セッション起動/停止、Beaconチャット、モバイル表示）

## スコープ外

- サイドバーの折りたたみ/リサイズ機能（初回リリース後に検討）
- サイドバーでのドラッグ&ドロップによるセッション並び替え
- セッションのグルーピング（リポジトリ別等）
